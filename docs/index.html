Problem Description

Here you should provide the context for your idea. Describe the problem that you are trying to solve, why it is important, where it is challenging. Give us a general idea on how you are going to solve it.

We want to solve the problem of planet generation. Given the texture for a planet, we want to procedurally generate a realistic looking planned, with terrain deformations at large and small scales. This is important because it allows for realistic looking worlds at multiple scales, which is especially useful in games such as open world games. One challenge is procedurally generating realistic looking terrain. We need to generate terrain that matches with the provided planet texture. For example the oceans should remain flat and maybe contain waves, whereas the land should have more significant bumps and craters. Another challenge is rendering the generated terrain at multiple scales. Ideally a planet should contain detail even when we zoom in. For example we want an earth size planet to contain not only mountain size bumps, but also plenty of distortions closer to the size of a person. However, generating an entire planet mesh with all these distortions would lead to an infeasibly large planet mesh. We intend to solve this issue by dynamically generating a mesh given our camera location and resolution, only computing the details for parts of the planet that will be visible to the camera.
